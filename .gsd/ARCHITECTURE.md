## Overview

DropZone is a modern, high-performance P2P file-sharing application. It utilizes **WebRTC** to establish direct browser-to-browser data channels, ensuring files never touch the server disk. A Node.js **Signaling Server** facilitates the initial handshake, while **DTLS 1.3** ensures all transfers are end-to-end encrypted.

## Components

### Frontend Main App
- **Purpose:** Handles drag-and-drop, UI state (Glassmorphism), room management, and reassembly of received chunks.
- **Location:** `index.html`, `script.js`, `animations.js`, `styles.css`
- **Dependencies:** `socket.io-client`

### P2P Engine
- **Purpose:** Manages RTCPeerConnection, ICE candidate negotiation (via STUN/TURN), and chunked RTCDataChannel transfers.
- **Location:** `webrtc.js`
- **Security:** DTLS 1.3 with ECDSA certificates.

### Signaling Server
- **Purpose:** Facilitates P2P handshakes by relaying Offers, Answers, and ICE candidates. Manages transient in-memory rooms.
- **Location:** `server.js`
- **Dependencies:** `express`, `socket.io`, `nodemailer` (for invitations).

## Data Flow

1. **Room Creation:** Sender creates a room via `/api/create-room`. Signaling server stores room metadata (file list) in-memory.
2. **Discovery:** Receiver joins via `shareUrl`. Signaling server notifies Sender of the new peer.
3. **Handshake:** Sender and Receiver exchange SDP Offers/Answers and ICE Candidates via Socket.io.
4. **Encryption:** Peers negotiate a secure DTLS 1.3 channel using ECDSA certificates.
5. **Transfer:** Files are read as ArrayBuffers, chunked (16KB), and streamed via `RTCDataChannel`# Technology Stack
